梯度下降

让机器自己学会调参

引言：从模型训练说起

一个最简单的线性回归模型可以写成：

$$ \hat{y} = wx + b $$

其中 $w$ 是权重（斜率），$b$ 是偏置（截距）。
训练模型的本质，就是搜索一组最优秀的 $w$ 和 $b$。

训练闭环：输入数据 → 预测 → 计算损失 → 计算梯度 → 更新参数 ...

损失函数与调参方向

损失函数用来度量模型预测值 $\hat{y}$ 与真实值 $y$ 之间的偏差。以均方误差（MSE）为例：

$$ L = MSE = \frac{1}{n}\sum_{i=1}^{n}(\hat{y}_i - y_i)^2 $$
  • 损失函数能告诉我们当前参数有多不准。训练的目标：让损失函数的值尽可能小。
  • 问题：如何决定是把参数调大一点，还是调小一点？

💡 我们需要一种自动调参的算法——梯度下降（Gradient Descent）。

数学基础：导数

导数：函数在某点的变化率

对于单变量函数 $f(x)$，它在 $x_0$ 处的导数定义为：

$$ f'(x_0) = \lim_{\Delta x \to 0} \frac{f(x_0 + \Delta x) - f(x_0)}{\Delta x} $$
  • 几何意义：函数曲线在该点的切线斜率。
  • $f'(x_0) > 0$：递增（$x$ 增大，$f(x)$ 增大）。
  • $f'(x_0) < 0$：递减（$x$ 增大，$f(x)$ 减小）。
导数告诉我们函数值随自变量变化的趋势。

数学基础：偏导数

偏导数：多变量函数的变化率

损失函数通常依赖于多个参数（如 $w$ 和 $b$）。我们需要固定其他变量不变，只对其中一个变量求导。

举个例子：$f(x, y) = x^2 + 3xy + y^2$

  • 对 $x$ 的偏导数：$\frac{\partial f}{\partial x} = 2x + 3y$ （把 $y$ 当常数）
  • 对 $y$ 的偏导数：$\frac{\partial f}{\partial y} = 3x + 2y$ （把 $x$ 当常数）
偏导数告诉我们每个参数对函数值的影响。

梯度下降核心原理

把所有变量的偏导数组合成一个向量，就得到了梯度（Gradient）。

$$ \nabla L(w, b) = \left(\frac{\partial L}{\partial w},\ \frac{\partial L}{\partial b}\right) $$

梯度有一个极其重要的性质：

  • 梯度的方向，是函数值增长最快的方向。
  • 梯度的反方向，就是函数值下降最快的方向。

梯度下降的迭代公式

基于梯度的性质，参数更新公式为：

$$ \boldsymbol{\theta}_{new} = \boldsymbol{\theta}_{old} - \eta \cdot \nabla L(\boldsymbol{\theta}_{old}) $$
  • $\boldsymbol{\theta}_{old}$：当前位置
  • $\nabla L(\boldsymbol{\theta}_{old})$：脚下最陡的上坡方向
  • $\eta$：学习率（Learning Rate），控制每一步迈多大
  • $-\eta \cdot \nabla L$：朝下坡方向迈一步

为什么要减去梯度？因为我们要最小化损失函数，所以需要往反方向走。

线性回归的梯度推导

损失函数（MSE）：

$$ L(w, b) = \frac{1}{n}\sum_{i=1}^{n}(wx_i + b - y_i)^2 $$

分别对 $w$ 和 $b$ 求偏导（链式法则）：

$$ \frac{\partial L}{\partial w} = \frac{2}{n}\sum_{i=1}^{n}(wx_i + b - y_i) \cdot x_i \quad \quad \frac{\partial L}{\partial b} = \frac{2}{n}\sum_{i=1}^{n}(wx_i + b - y_i) $$

手动实现梯度下降

使用 Python 和 NumPy 编写完整的梯度下降算法，并记录训练过程： 

def gradient_descent(X, y, lr=0.1, epochs=200):
    w, b = 0.0, 0.0
    n = len(X)
    history = {'w': [w], 'b': [b], 'loss': [np.mean((w*X + b - y)**2)]}
    
    for epoch in range(epochs):
        y_pred = w * X + b
        
        dw = (2/n) * np.sum((y_pred - y) * X)
        db = (2/n) * np.sum(y_pred - y)
        
        w -= lr * dw
        b -= lr * db
        
        history['w'].append(w)
        history['b'].append(b)
        history['loss'].append(np.mean((y_pred - y)**2))
        
        if epoch % (epochs/100) == 0:
            print(f"迭代 {epoch:3d}: w={w:.2f}, b={b:.2f}, loss={history['loss'][-1]:.2f}")
            
    return history

history = gradient_descent(areas, prices, lr=0.01, epochs=100000)

学习率 $\eta$：最关键的超参数

我们在 $f(x)=x^2$ 上进行实验，直观感受不同学习率的影响： 

def gradient_descent_1d(start_x, lr, n_steps=15):
    """在 f(x)=x² 上做梯度下降"""
    trajectory = [start_x]
    x = start_x
    for _ in range(n_steps):
        grad = 2 * x                # f'(x) = 2x
        x = x - lr * grad           # 参数更新
        trajectory.append(x)
        if abs(x) > 500:            # 已经爆炸，提前停
            break
    return trajectory

学习率实验：损失曲面轨迹

在 $y = x^2$ 中，更新公式是：$x_{new} = x - lr \times 2x = x(1 - 2 \cdot lr)$
学习率的选择通常需要实验调整，常见的起始值有 0.01、0.001、0.0001 等。

梯度下降的优化算法

标准梯度下降存在收敛慢、易陷入局部最优等局限。业界发展出了多种优化器：

  • 动量法（Momentum）：借鉴物理学惯性，给梯度加上“速度”，减少狭长山谷中的震荡。
  • AdaGrad：为每个参数自动维护独立的学习率，适合处理稀疏数据。
  • RMSProp：AdaGrad 的改良，关注近期的梯度变化，避免学习率降到零，适合非平稳目标（如 RNN）。
  • Adam（自适应矩估计）：结合了动量（方向加速）和 RMSProp（自适应步幅）。它同时维护梯度的方向趋势和大小变化，既跑得快又稳。收敛快且对学习率不敏感，是目前深度学习中最广泛使用的默认首选优化器。

Adam 优化器：代码实现

Adam = Momentum + RMSProp。同时维护梯度的方向趋势（一阶矩）和大小变化（二阶矩）。 

def adam_optimizer(X, y, lr=0.1, epochs=200, beta1=0.9, beta2=0.999, epsilon=1e-8):
    w, b = 0.0, 0.0
    m_w, v_w = 0.0, 0.0  # 一阶矩和二阶矩
    m_b, v_b = 0.0, 0.0
    
    for epoch in range(epochs):
        t = epoch + 1
        y_pred = w * X + b
        dw = (2/len(X)) * np.sum((y_pred - y) * X)
        db = (2/len(X)) * np.sum(y_pred - y)
        
        # 更新一阶矩估计和二阶矩估计
        m_w = beta1 * m_w + (1 - beta1) * dw
        v_w = beta2 * v_w + (1 - beta2) * (dw**2)
        # 偏差校正
        m_w_hat = m_w / (1 - beta1**t)
        v_w_hat = v_w / (1 - beta2**t)
        
        # 更新参数
        w -= lr * m_w_hat / (np.sqrt(v_w_hat) + epsilon)

梯度下降的变体

每次更新参数时，该用多少数据来计算梯度？

变体 数据量 特点
全量 (BGD) 全部数据 梯度精确，稳定，但计算开销极大。
随机 (SGD) 1 条数据 轻量，有随机性，但噪声大、震荡剧烈。
小批量 (Mini-batch) 一小批 (如 64) 折中方案。能利用 GPU 并行，兼顾精度与效率。

Mini-batch 已经成为深度学习训练的业界主流选择。

梯度下降是深度学习的基础

从最经典的神经网络到 GPT 等大语言模型，无一例外都依靠梯度下降来训练和优化。理解了梯度下降，就打通了理解一切深度学习模型的底层逻辑。

训练的核心引擎

  • 前向传播：计算输出和损失值
  • 反向传播：利用链式法则计算梯度
  • 梯度下降：拿到梯度后更新参数
$$ \text{深度学习训练} = \text{反向传播} + \text{梯度下降} $$

基本范式从未改变

GPT 的训练本质上与线性回归完全一致，只是规模更大：

  • 模型更复杂（Transformer）
  • 参数更多（数千亿）
  • 数据更大（整个互联网）

核心思想：沿着梯度方向更新参数，让损失越来越小。

总结

  • 损失函数：度量模型预测值与真实值之间的差距，是优化的目标。
  • 梯度：所有偏导数组成的向量，指向函数增长最快的方向。
  • 梯度下降公式：$\theta_{new} = \theta_{old} - \eta \cdot \nabla L$
  • 学习率：控制每一步的步幅，太大震荡，太小太慢。
  • Adam 优化器：结合动量和自适应学习率，是当前最常用的优化器。
  • Mini-batch：折中全量和单样本，是实际训练中的主流方式。
  • 反向传播 + 梯度下降：深度学习训练的核心引擎。
