# 生成 HTML 幻灯片

## 概述
生成结构清晰、可直接演示的 HTML 幻灯片，强调排版层级、投屏可读性与键盘翻页体验。除非用户明确要求多文件工程，默认产出单个自包含 `.html` 文件。

## 工作流
1. 明确输入信息和输出目标。
2. 搭建或更新幻灯片结构。
3. 应用 TailwindCSS 视觉系统（主题色、字号、间距、动效）。
4. 补充导航能力、讲者备注和 SVG 配图。
5. 在投屏与移动端视口下做可读性校验。

## 前置信息

- 受众：有编程基础的计算机专业研究生
- 目标页数：尽量 15 页以内。
- 语气风格：正式、专业。
- 输出路径与文件名：尽可能与输入文件名相关。

## 编写规则
- CSS 统一使用 TailwindCSS。
- 单文件产出场景下，优先用 Tailwind Play CDN + 内联 `tailwind.config`。
- 每页使用一个 `<section class="slide">`，每页只表达一个核心信息点。
- 优先短句和层级，不要堆积长段文字。
- 保证投影场景下对比度与字号充足。
- 图片或图形优先使用内联 SVG，避免外部图片依赖。
- 代码示例需避免溢出，必要时换行。

## 投屏可读性最低标准
- 标题建议不小于 `text-5xl`（或约 48px）。
- 正文建议不小于 `text-2xl`（或约 24px）。
- 单行字数控制在 18-28 个中文字符附近。
- 重要图形元素（图标/图表）最小尺寸不低于 240x140。
- 在 `1920x1080` 投屏下优先检查可读性，再检查 `1366x768`。

## 导航与备注
至少实现以下键盘导航：
- `ArrowRight` / `PageDown`：下一页。
- `ArrowLeft` / `PageUp`：上一页。
- `Home`：跳到第一页。
- `End`：跳到最后一页。

讲者备注可选两种方式：
- 每页内嵌 `<aside class="notes">...</aside>`。
- 若需更干净 DOM，可使用 HTML 注释。

## 快速脚本
使用 `scripts/scaffold_slides.py` 先生成基础骨架：

```bash
python3 scripts/scaffold_slides.py \
  --output deck.html \
  --title "机器学习项目阶段汇报" \
  --slide "任务定义|业务目标：降低流失率\\n预测窗口：未来 30 天\\n评估指标：AUC 与召回率" \
  --slide "数据与特征|样本量：120 万\\n核心特征：近 7 天活跃、支付频次、客服交互\\n数据问题：类别不平衡" \
  --slide "模型评估与迭代|基线：Logistic Regression AUC=0.73\\n当前：XGBoost AUC=0.81\\n下一步：阈值优化与线上 A/B"
```

生成后再按实际内容手动打磨文案、版式与视觉细节。

## 质量标准
交付前至少确认：
- 各页标题唯一且易扫读。
- 在 `1920x1080` 下无文字重叠、无图形过小。
- 动效克制，且只服务于信息表达。
- 页面不依赖外部图片资源，配图尽量为内联 SVG。
- 末页包含总结与下一步行动。
