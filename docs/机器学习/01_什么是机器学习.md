## 机器学习究竟是什么
要想定义机器学习，首先需要了解一个更常见的术语：人工智能

### 什么是人工智能
人工智能（Artificial Intelligence，AI）是计算机科学的一个分支，致力于创建能够模拟、延伸和扩展人类智能的理论、方法、技术及应用系统。简单来说，就是**让机器具备感知、学习、推理和决策的能力**。

在日常语境中，人工智能一词往往有双重含义——它既可以代指实现机器智能的方法与理论，也可以指呈现出人类智能特征的应用载体（如 ChatGPT、智能机器人等）。

### 机器学习是人工智能主要的实现路径
机器学习（Machine Learning，ML）是人工智能的一个分支，研究如何让计算机在**无需显式编程的情况下，通过数据自动学习规律**，对未来可以预测。

机器学习是当前实现人工智能最核心、最有效的方法，尽管广义的人工智能涵盖了早期基于人工规则的专家系统，但当代几乎所有前沿的 AI 应用体系，都建立在机器学习的方法论之上。

我们在日常生活中会根据经验对未来做预测，比如我告诉你，我戒烟了、正在多吃蔬菜和锻炼身体，你预测一年后我的身体健康状况会发生什么变化？也许会有所改善。

发现数据背后的规律和相关性是人类的常识，这也正是机器学习的意义之所在，唯一的区别是机器学习由计算机来完成。

在机器学习之前如果我们想让计算机完成一个任务，就必须编写一个程序，清楚的描述每个步骤的执行过程，这种只适合简单的任务，有些任务对于这个范式来说过于复杂。例如，识别一个图像中是否包含苹果的任务。如果我们试图通过编程完成这个任务，很快会发现这是一件很难的事情。

但人类是如何知道什么是苹果的？我们从未背诵过苹果的精确定义，更不知道它的几何参数、纹理方程或颜色阈值。

在反复见过无数次苹果（实物、照片、视频）之后，我们的大脑会自动从海量样本中提取关键特征：轮廓的弧度、表皮的光泽、果柄的位置、切面的纹理等。神经元的连接在不断的刺激中自行调整，最终在脑海中形成了一个模糊却精准的概念——“苹果”。

在机器学习中这就是我们让计算机做的事情，我们向计算机展示多张图像，并且哪些图像是包含苹果，重复这个过程直到计算机捕捉到构成苹果的关键模式和特征。这时向计算机展示一张新的图像，它可以使用学习到的模式来确定图像中是否包含苹果。

当然，我们还需要编写让计算机捕获模式的程序，但我们并没有编写其答案，只是在告诉计算机应该如何学习。

### 深度学习是用了神经网络机构的机器学习
现在我们也可以回答什么是深度学习了。在机器学习的过程中可以使用多种技术让计算机学习到数据背后的模式，其中一种**使用了神经网络架构**的技术表现非常出色，因此形成了自己的研究领域，称之为深度学习（Deep Learning，DL）。

深度学习本质上仍然是机器学习，它同样遵循从数据中学习规律的基本范式，只是在模型结构上采用了多层神经网络，使其能够自动学习更深层次、更抽象的特征表示。

传统机器学习识别一张猫的图片，需要人工告诉模型去关注哪些特征（轮廓、纹理、耳朵形状……），然后模型基于这些特征做出判断。深度学习直接将原始像素输入神经网络，模型自行学会从边缘→纹理→局部结构→整体语义的逐层抽象，最终做出判断。

传统机器学习擅长处理结构化数据，不需要极其庞大的算力，在金融风控、商品推荐等领域依然非常有用。而深度学习处理非结构化数据（图片、视频、语音、自然语言文本），但需要极其海量的数据和庞大的算力，广泛应用于人脸识别、自动驾驶、大语言模型等领域。

### 三者的关系

简单来讲，深度学习 ⊂ 机器学习 ⊂ 人工智能

+ **人工智能**是最宽泛的概念，其核心理念是让机器表现出类似人类的智能；
+ **机器学习**是人工智能的一个子集，其核心理念是不靠人工编写规则，让机器自己从数据里找规律；
+ **深度学习**特指使用了多层神经网络架构的机器学习，可以自动提取数据特征，发现更深层的模式。

## 如何让机器根据数据做出决策
计算机根据数据做决策，背后的工作原理和人类根据经验做出决策的过程非常相似

![](https://cdn.nlark.com/yuque/0/2026/png/87727/1771637114302-1c6383f9-f6ba-4dcc-8cfa-a60616ad7353.png)

### 人类是如何思考的
人类需要根据经验作出决策时候，通常使用以下框架

1. 记住过去相似的情况
2. 制定一个通用的规律
3. 使用这个规律来预测未来可能要发生的事情

如果问题是“今天会下雨吗？”，猜测的过程如下：

1. 最近一年有乌云的大部分日子都下雨了
2. 有乌云大概率就要下雨
3. 今天会下雨

这个预测可能是对的，也可能不对，但至少我们正在努力根据掌握的信息做出准确的预测。

### 机器学习的术语——模型与算法
在机器学习中，我们让计算机通过分析数据来解决实际问题，其核心手段就是构建模型。那么究竟什么是模型？

从广义上讲，模型是对现实世界复杂事物的简化与抽象。我们建立模型，是为了更清晰地理解、分析、预测或控制那些错综复杂的现象。而在机器学习领域，模型的本质是计算机通过学习大量数据后，自主总结出的一套**规则集合**，用于对未来的新数据进行预测或决策。

回顾前文预测下雨的例子：在这个场景中，模型就是对现实天气规律的一种极简表示。在这个被简化的世界里，只提炼出了一条核心规则——出现乌云即意味降雨。

在实际应用中，很多人容易将“算法”与“模型”混为一谈。简单来说：在机器学习中，**算法是学习的方法，而模型是学习的结果**。  
在上述预测下雨的例子中，算法的作用就是指导计算机去观测下雨天的气象，并找出共同特征——乌云。当然，真实的机器学习算法远比这复杂得多，但它们都是由一系列特定的逻辑步骤组成。我们可以这样定义：算法是用于解决问题或执行计算的一套明确步骤；而在机器学习的语境下，我们选择使用的算法目标是从数据中构建出一个模型。

**简而言之：算法负责构建模型，而模型负责预测未来。**

### 人类使用模型
什么是机器学习？在深入探讨复杂的算法之前，我们先来看看人类在日常生活中是如何建立模型的。

让我们来看一个经典的应用场景——垃圾邮件识别。

**示例 1：一个烦人的邮件狂人朋友**

我们的朋友小张热衷于发邮件，但他的很多邮件都是垃圾邮件，这着实让人有些头疼。今天是星期六，我们刚收到小张的一封新邮件。有没有办法在不打开邮件正文的情况下，就判断出这是不是垃圾邮件？

为了解决这个问题，我们在脑海中回想了一下小张最近发来的 10 封邮件。这就是我们的历史数据。在这 10 封邮件中，有 6 封是垃圾邮件。基于这个信息，我们可以总结出第一个粗略的模型：

> 模型 1.0：小张每发送 10 封邮件，就有 6 封垃圾邮件。


现在，我们有了一个简单的规则（模型），可以用来预测小张的新邮件了。从概率上看，这封新邮件有 60% 的可能是垃圾邮件，40% 的可能不是。因为它是垃圾邮件的概率更高，所以我们预测这封邮件是垃圾邮件。

虽然这个预测有可能是错的，但鉴于我们手头只有这 10 封邮件的数据，这是我们目前能做出的最佳猜测。但是我们还能做得更好吗？

**示例 2：周期性烦人的电子邮件**

在模型 1.0 中，我们对小张的所有邮件一视同仁，仅仅依靠“历史概率”来盲猜。但实际上，邮件里往往隐藏着更多信息。让我们尝试对邮件进行更深入的分析，比如看看邮件的发送时间，看看能不能找到新的规律。

我们仔细查看了小张上个月发送邮件的日期，发现了一丝端倪：

| **发送日期** | **分类** |
| --- | --- |
| 星期一 | ✅ 非垃圾邮件 |
| 星期二 | ✅ 非垃圾邮件 |
| 星期日 | ❌ 垃圾邮件 |
| 星期五 | ✅ 非垃圾邮件 |
| 星期六 | ❌ 垃圾邮件 |
| 星期六 | ❌ 垃圾邮件 |
| 星期三 | ✅ 非垃圾邮件 |
| 星期四 | ✅ 非垃圾邮件 |
| 星期日 | ❌ 垃圾邮件 |
| 星期三 | ✅ 非垃圾邮件 |


规律似乎很明显：小张在工作日发送的都是正常邮件，而周末发送的全是垃圾邮件！基于这个新发现，我们制定了一个更有依据的模型：

> 模型 2.0：小张在工作日发送的是正常邮件，在周末发送的是垃圾邮件。


根据这个新模型，因为今天是周末，我们可以非常自信地预测这封新邮件是垃圾邮件，看都不用看直接扔进垃圾箱。

**示例 3：事情变复杂了**

我们开心地使用着模型 2.0。直到有一天遇到小张，他疑惑地问：“你上周末怎么没来参加徒步活动？”，我们一头雾水。原来他上周末发了一封活动通知邮件，却被我们的“模型 2.0”当成垃圾邮件过滤掉了。

看来只看“发送时间”还是太绝对了，我们需要一个更完善的模型。让我们回头再看看小张的数据，这次我们关注一下邮件的**文件大小**：

| **邮件大小** | **分类** |
| --- | --- |
| 1 KB | ✅ 非垃圾邮件 |
| 2 KB | ✅ 非垃圾邮件 |
| 16 KB | ❌ 垃圾邮件 |
| 3 KB | ✅ 非垃圾邮件 |
| 20 KB | ❌ 垃圾邮件 |
| 18 KB | ❌ 垃圾邮件 |
| 3 KB | ✅ 非垃圾邮件 |
| 5 KB | ✅ 非垃圾邮件 |
| 7 KB | ✅ 非垃圾邮件 |
| 35 KB | ❌ 垃圾邮件 |


我们又发现了一个规律：超过 10 KB 的邮件往往是垃圾邮件！这很符合逻辑，因为垃圾邮件通常带有塞满广告的大附件。结合之前的发现，我们可以把模型升级一下：

> 模型 3.0：如果邮件是周末发送的，且大小超过 10 KB，则是垃圾邮件；否则是正常邮件。


使用这个新模型后，我们既拦截了周末的广告，又成功收到了只有 3 KB 大小的周末徒步通知。

### 机器学习使用模型
在上面的故事中，为什么我们预测的准确率越来越高？因为我们逐渐利用了“发送时间”和“邮件大小”这两个信息。在机器学习中，这些可用于预测数据的属性被称为**特征（Feature）**。

想象一下，如果我们引入更多判定特征，比如邮件标题是否包含中奖或免费、是否包含大量的感叹号、发件人地址是否在通讯录等，我们的预测会变得极其精准。

人类的直觉一次只能处理两三个特征，几十条数据；而计算机可以不知疲倦地快速分析数百万封邮件和成千上万个特征，从而建立一个极其强大且复杂的预测模型。

在机器学习中，算法可能会为不同的特征分配不同的**权重（Weight）**，最终得出一个数学公式。以垃圾邮件分类器为例，它的底层逻辑可能类似这样：

> 模型 4.0：得分 = (权重A × 发送时间是否周末) + (权重B × 邮件大小) + (权重C × 是否包含“免费”) + ...


_**例如：**_

+ 如果包含“免费”，权重 C 非常高，得 50 分；
+ 如果邮件大于 10KB，权重 B 较高，得 30 分；
+ ...
+ 如果总得分超过 60 分阈值 ➔ 判定为垃圾邮件。

在这个公式中，计算机需要解决的核心问题是：如何精准地确定每一个特征的权重？ 权重给高了容易误判正常邮件，给低了又漏掉垃圾邮件。

**让计算机通过大量的历史数据，自己去不断调整和寻找这组最优权重的过程，就是机器学习，**也正是我们接下来要深入探讨的主要内容。

## 机器学习的学前准备
很面对机器学习庞大的知识体系，很多初学者常会感到望而却步。请放心，虽然掌握一定的数学知识和编程技能是必需的，但你完全不需要成为这些领域的专家。在接下来的学习中，每当涉及核心理论或代码实现时，都会提供 Just-in-Time 的通俗讲解，帮你扫清障碍。

下面就是整个课程涉及的全部前置知识——是的，就这么多

### 数学基础
+ **线性代数**：了解向量和矩阵基础概念，以及怎么做基础运算（加减、转置、点积和矩阵乘法）；
+ **概率与统计**：理解均值、方差、概率分布，以及经典的贝叶斯定理；
+ **微积分**：只需理解导数、偏导数的含义，以及什么是链式法则即可。

+ **常用库**：
    - **NumPy / Pandas**：数值计算与数据清洗、表格数据处理
